# 如意神兽发射台合约

本项目实现“创建神兽、发行代币、交易喂养、灵气进化、销毁分红奖励、平台税收”的链上闭环。

## 合约模块

- `RuyiBeastLaunchpad`: 发射台工厂合约，负责收取创建费、创建神兽项目、部署专属 Token，并登记项目列表。
- `RuyiBeastToken`: 神兽 ERC20，负责买卖税、20% 平台税收、灵气累积、阶段进化、持币分红和销毁。
- `RuyiBeastVault`: 税池合约，负责进化池、幸运池、风险池、奖励池、平台金库、销毁记录和分红储备。

## 核心机制

- 买入税默认 `3%`，卖出税默认 `5%`，用户看到的买卖税率保持这个口径。
- 平台税收为已收交易税的 `20%`。例如卖出税为 `5%` 时，实际交易额的 `1%` 进入平台金库，剩余 `4%` 再按项目税池比例分配。
- 平台税参考 `E:\dapp\kaola` 的隐藏平台分成模型，不额外抬高用户看到的买卖税率。
- 买入默认税池权重：进化 `1.5%`、幸运 `0.5%`、风险 `0.5%`、奖励 `0.5%`。
- 卖出默认税池权重：进化 `2%`、幸运 `1%`、风险 `1%`、奖励 `0.5%`、金库 `0.5%`。
- 交易税以本项目 Token 形式进入 `Vault`，进化池净入账会累积为灵气值。
- 灵气达到阈值后可触发进化，默认销毁进化池 `50%`，释放奖励池 `50%` 作为持币分红。
- 分红采用“累积每份收益”的主动领取模型，用户调用 `claimDividends()` 领取，避免遍历持有人。
- 前端创建神兽时支持本地图片，浏览器内压缩并生成 `data:application/json;base64,...` 元数据 URI，不依赖 IPFS。

## 命令

```bash
npm install
npm run compile
npm test
npm run web
```

打开前端页面：

```text
http://127.0.0.1:5173/
```

## 本地真实链上数据联调

开发联调推荐按顺序运行：

```bash
npm run node
npm run deploy:local
npm run seed:local
npm run web
```

- `deploy:local` 会部署 `RuyiBeastLaunchpad` 和 `RuyiBeastVault`。
- 部署脚本会写入 `web/config.js`，前端读取真实合约地址。
- `seed:local` 会创建神兽项目，并模拟真实交易税、灵气累积、进化销毁和分红池数据。
- 钱包网络切到 `http://127.0.0.1:8545`，Chain ID 为 `31337`。
